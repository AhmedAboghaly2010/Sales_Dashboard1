# 📊 لوحة تحليل المبيعات التفاعلية
## Sales Analytics Dashboard

تطبيق ويب تفاعلي لتحليل بيانات المبيعات وعرضها في داش بورد احترافي مع دعم كامل للغة العربية.

---

## 🚀 رفع التطبيق على الإنترنت (مجاناً)

### الطريقة الأسهل: Streamlit Community Cloud

1. **أنشئ حساب على GitHub** (مجاناً): https://github.com
2. **أنشئ Repository جديد** واسمه `sales-dashboard`
3. **ارفع الملفات التالية** في الـ Repository:
   ```
   app.py
   requirements.txt
   .streamlit/config.toml
   ```
4. **اذهب إلى**: https://share.streamlit.io
5. **سجل الدخول** بحساب GitHub
6. اضغط **"New app"** واختر الـ Repository
7. في **"Main file path"** اكتب: `app.py`
8. اضغط **"Deploy!"**

✅ التطبيق هيبقى متاح على رابط مثل:
`https://your-name-sales-dashboard.streamlit.app`

---

## 💻 تشغيل محلياً

```bash
# تثبيت المكتبات
pip install -r requirements.txt

# تشغيل التطبيق
streamlit run app.py
```

افتح المتصفح على: http://localhost:8501

## 📊 مميزات التطبيق

| الميزة | الوصف |
|--------|-------|
| 📁 رفع الملفات | يقبل Excel (.xlsx/.xls) و CSV |
| 🤖 كشف تلقائي | يكشف أعمدة التاريخ والكمية والتصنيف تلقائياً |
| 📊 Tab 1 | مقارنة أعلى N تصنيف (عمودي / دائري) |
| 📅 Tab 2 | الاتجاه الزمني سنوياً وشهرياً |
| 🥧 Tab 3 | التوزيع النسبي لكل تصنيف |
| 🔥 Tab 4 | خريطة التقاطع (Heatmap) بين التصنيفات |
| 📋 Tab 5 | عرض البيانات الخام مع فلترة وتحميل |
| 💾 تصدير | تحميل البيانات المفلترة كـ Excel |

---

## 🗂️ شكل البيانات المطلوبة

يجب أن يحتوي ملفك على:

| العمود | الوصف | مثال |
|--------|-------|-------|
| `Billing Date` | تاريخ الفاتورة | 2024-01-15 |
| `Billed Quantity` | الكمية | 150 |
| `MG1`, `MG2`, ... | أعمدة التصنيف | فئة أ، فئة ب |
| `Amount` (اختياري) | قيمة المبيعات | 5000 |

> **ملاحظة:** يمكنك تغيير اسم أي عمود من الشريط الجانبي في التطبيق

---

## 📞 الدعم

في حال وجود مشكلة، تأكد من:
- أن ملفك يحتوي على صف عناوين (headers) في السطر الأول
- أن أعمدة التاريخ بصيغة `YYYY-MM-DD` أو `DD/MM/YYYY`
- أن أعمدة الكمية تحتوي على أرقام فقط
